# redanalyst

Настройка и аудит **веб- и сквозной аналитики** для Claude Code. Основной стек — **Яндекс.Метрика +
Яндекс.Директ** (вторичный GA4/GTM). Ведёт проект от «стоит ли счётчик» до «вижу ROI по кампаниям»:
работает через API (читать конфиг, создавать цели, грузить офлайн-конверсии, читать расход) и выдаёт
инструкции человеку там, где API не хватает (доступ к кабинету, вставка сниппета, 152-ФЗ согласие).

Standalone-скилл (без code-dep на `core/`), как `redjob`/`redbrain`. Runtime-deps (`deps.txt`) —
`plan-panel`, `redloft`, `redresearch`, `redjob` — ставятся отдельно, вызываются по мере надобности.

## Ядро: «настроено» ≠ «работает»

Главный принцип (выстрадан на проде, см. `references/case-ecommerce-1c.md`): проверять эмпирически,
по источнику истины, а не по прокси. «0 конверсий в Метрике» ≠ «0 продаж» (истина у эквайера/CRM/1С);
HTTP 200 от Offline API ≠ конверсия засчитана (`uploading/{id}=processed` + строка в отчёте).

## Режимы

| Команда | Что делает |
|---|---|
| `/redanalyst-audit <домен\|counter_id>` | что подключено и **работает** → gap-report с приоритетами |
| `/redanalyst-setup` | настроить по gap-листу (счётчик, цели, ecommerce, офлайн, Директ, UTM) |
| `/redanalyst-goals <бизнес>` | ресёрч воронки → какие цели/KPI под ЭТОТ бизнес |
| `/redanalyst-ecom` | выручка: ecommerce-события ИЛИ серверные офлайн-конверсии |
| `/redanalyst-crosschannel` | 3 ноги (расход+визиты+выручка) → ROI/ROMI/CPA по кампаниям |
| `/redanalyst-verify` | данные **реально** идут (факт в отчёте, не «200 от API») |

## Безопасность

- Любая запись в живой счётчик — по write-safety contract: `GET-dedup → preview → явное «да» →
  execute → read-back` + kill-switch + dry-run→canary. `lib/metrika.sh` — dry-run по умолчанию.
- Токены Метрики/Директа — только через 1Password `op run`, `scope: proj-<slug>`, никогда в чат/лог.
- Идемпотентность офлайн-загрузок — `lib/ledger.py` (SQLite, `purchase_id` PK) → без двойной выручки.
- Артефакты local-first, вне облачных папок; PII-purge.

Полный контракт — `SKILL.md` + `_shared.md` + `commands/`. Доменные знания — `references/`.
