# /redanalyst-verify — данные РЕАЛЬНО идут

Вес: lightweight (read-only). Сердце CORE PRINCIPLE. Отвечает не «сделали ли вызов», а «есть ли факт
в отчёте / у источника истины». Гоняется после setup/ecom и периодически.

## Что проверяет (каждый пункт — фактом, не галочкой)
1. **Цели срабатывают**: `ym:s:goal<id>reaches` за период > 0 для целей, что должны срабатывать. Ноль
   у боевой цели → сломано (не тип? не то имя события? счётчик не на странице?).
2. **Офлайн-конверсии**:
   - `GET uploading/{id}` последнего батча = `processed` (не «отправили»).
   - Построчный диагностический отчёт: сколько привязалось, сколько LINKAGE_FAILURE и почему.
   - Строки видны в отчёте по цели (учесть лаг ~2ч, иногда 24–48ч).
   - **Сверка с источником истины**: число загруженных конверсий ≈ число реальных заказов с ClientID
     в окне (из 1С/CRM). Расхождение = «настроено, но не течёт» → расследовать сверху вниз.
   - Дубли: ledger не содержит повторов `purchase_id`; revenue в отчёте не задвоен.
3. **Директ**: расход виден в «Источники, расходы и ROI» / Reporting API (не «№ вписан»).
4. **Крон** (если офлайн-выгрузка): подтверждён в планировщике хоста (Beget `cron/getList`), не
   `crontab -l`. См. `references/ops-gotchas.md`.
5. **ClientID-цепочка**: тестовый заказ → ClientID доехал до записи заказа/CRM/1С (не только в форме).
   См. `references/clientid-capture.md`.

## При расхождении — расследовать сверху вниз (как в кейсе-эталоне)
«0 в отчёте» → сверь с источником истины (были ли продажи?) → если были, иди по цепочке: сайт забрал
факт оплаты? ClientID доехал? в окне? загрузка `processed`? привязка не LINKAGE_FAILURE? Каждое звено —
факт. Не останавливаться на первом «вроде норм».

## Done-when
`verify-report.md`: по каждому пункту — ✅ подтверждён фактом / ⚠️ расхождение (с гипотезой звена) /
⛔ сломано (что и где). Суммы/ClientID маскированы (`_shared.md §Secrets`/PII). `state.goals[].verified`
и `last_batch.status` обновлены.

## Вывод
Короткий статус «течёт / где рвётся». Если рвётся — конкретное звено и следующий шаг (какой режим/что
человеку). Не «200 от API» — а факт.
