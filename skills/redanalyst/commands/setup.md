# /redanalyst-setup — настроить по gap-листу

Вес: **phased** (write). Идёт ПОСЛЕ аудита (нужен `gap_list`). Нетривиальный план — через `plan-panel`
до записи (как в кейсе-эталоне: 2× панель). Токены: `metrika:write` + офлайн-грант по мере надобности,
`scope: proj-<slug>`.

## Предусловия
- Есть свежий `state.json` с `gap_list` (нет → сначала `/redanalyst-audit`).
- `REDANALYST_KILL` не выставлен. Write идёт по **write-safety contract** (`_shared.md §3`).

## Фазы (по gap-листу, каждая — checkpoint в state)
1. **Счётчик** (если нет): инструкция человеку на установку тега + включение «Загрузка данных». Скилл
   не вставляет код на боевой сайт сам — выдаёт сниппет и место вставки, проверяет по факту.
2. **Цели**: взять `goals-spec.md` из `/redanalyst-goals` (или собрать воронку + micro/macro прямо
   тут). Для каждой цели: `GET-dedup` (нет ли уже) → PREVIEW payload → «да» → `POST goals` → **read-back**
   (`GET goals` + для action-цели проверить срабатывание). Записать `target_id` в state.
3. **Ecommerce dataLayer** (если магазин): инструкция/сниппет `purchase`+`items[]`; проверить событие.
4. **Офлайн-конверсии**: делегировать в `/redanalyst-ecom` (там ledger, canary, матчинг).
5. **Связка Директа**: инструкция человеку (№ счётчика в кампании, права) → verify **расход виден**.
6. **UTM-стратегия**: единая схема source/medium/campaign; выдать шаблоны меток.

## kill-switch / безопасность
Любая запись в живой счётчик — только после явного «да» на конкретный payload. Никогда не трогать
платёжный статус заказа (отдельное поле под YM). dry-run по умолчанию.

## Done-when
Каждый закрытый gap помечен `status:done` в state ТОЛЬКО после факта (цель в `GET goals` +
срабатывает; расход Директа виден; ecommerce-событие поймано). `setup-runbook.md` фиксирует что
сделано и что осталось человеку.

## Handoff
Собрать блок для `/rc-sync --infra` (`_shared.md §8`) и предложить прогнать — чтобы подключённая
аналитика задокументировалась во всех слоях проекта. Если проект со сквозной — предложить джобу
непрерывного self-check (redjob, `_shared` + SKILL §self-check), не активируя без «да».
