# Связка Яндекс.Директ ↔ Метрика + расход

Нога «расход» в сквозной аналитике. Без связки Директа с Метрикой отчёт «Источники, расходы и ROI»
не посчитает ROI, а визиты из рекламы не разметятся.

## Как связать (инструкция человеку — делается в кабинетах)

1. В настройках **кампании Директа** указать № счётчика Метрики (одна кампания может слать в счётчик).
2. Убедиться, что аккаунты Директа и Метрики имеют общий доступ (один логин-владелец либо выданные
   права представителя). Это шаг человека — доступ к кабинету скилл не выдаёт.
3. **yclid-автопометка** включена всегда, отключить со стороны Директа нельзя — она автоматически
   добавляет `yclid` к ссылкам объявлений. Это fallback-склейка клик↔визит даже без UTM.

## Что проверить на аудите (не «связано ли», а «виден ли расход»)

- Отчёт Метрики **«Источники, расходы и ROI»** показывает расход по кампаниям Директа за период? Если
  пусто при активных кампаниях — связка не доехала.
- Reporting API: расход тянется параметром `direct_client_logins`. Пустой результат при живых
  кампаниях = связка/права не настроены.
- Визиты с `utm_source=yandex` / наличием `yclid` присутствуют? (Logs API / отчёт по источникам.)

## Не-Яндекс расход — Costs import API («Расходы»)

Для VK Ads, Telegram Ads, таргета и пр. автосвязки нет. Расход грузится вручную:
- Management API, загрузка расходов по **UTM-разметке** (source/medium/campaign) + суммы за даты.
- Требует консистентной UTM-стратегии (см. `crosschannel-3legs.md` нога «б»), иначе расход не сойдётся
  с визитами.
- Так все каналы (Яндекс + не-Яндекс) оказываются в одном ROI-отчёте.

## Частые провалы

- Расход есть в кабинете Директа, но не в Метрике → № счётчика не указан в кампании ИЛИ нет прав.
- UTM в объявлениях не-Яндекс каналов кривые/отсутствуют → расход не матчится с визитами → канал
  проваливается в `(not set)`.
- Считают ROI, а расход по половине каналов не загружен → ROI занижен/ложен. См. precondition-guard в
  `crosschannel-3legs.md`: не показывать ROI как посчитанный, если расход по каналу неизвестен.
