# Захват ClientID — без него офлайн не привязать

ClientID — идентификатор посетителя в Метрике. Это ключ, которым офлайн-продажа приклеивается к
веб-визиту. Захватить его надо **в момент, когда пользователь ещё на сайте и создаёт заказ**; позже
восстановить нельзя (см. урок про backfill в `case-ecommerce-1c.md`).

## Как получить

```js
ym(COUNTER_ID, 'getClientID', function(clientID){
  // clientID — строка. ЗАПИСАТЬ в скрытое поле формы / прокинуть в заказ.
  document.querySelector('#ym_client_id').value = clientID;
});
```

**Грабли метода (все реальны):**
- `getClientID` **асинхронный** — значение доступно только внутри колбэка. Синхронно прочитать нельзя.
- До загрузки счётчика ID пустой → не дёргать до инициализации `ym`.
- Теряется при захвате ПОСЛЕ редиректа (новая страница — другой контекст). Захватывай на странице
  оформления, до перехода на «спасибо»/платёжку.
- Таймаут-guard (~300мс): если колбэк не вернулся, не блокировать оформление заказа — записать пусто и
  пометить заказ как «без ClientID» (он не попадёт в сквозную by design, это ок).

## Проброс front → back (самое хрупкое звено)

Значение надо донести **до самой записи заказа**, а не просто положить в форму:

```
hidden field (#ym_client_id)
   → POST оформления заказа
   → контроллер заказа (⚠️ whitelist полей!)
   → сохранение в заказ / отправка в CRM / 1С
```

**Грабля кейса:** ClientID терялся в **whitelist контроллера** — поле не было в списке
разрешённых, тихо отбрасывалось. Значение «есть в форме» ≠ «доехало до записи заказа». Проверяй
**сквозной проброс до самой записи**: создай тестовый заказ, посмотри, лежит ли ClientID в
сохранённом заказе / в CRM-карточке / в 1С, а не только в POST-теле.

## Web-only gate (согласовать с заказчиком)

Часть продаж физически не даёт ClientID:
- заказ по телефону/в оффлайн-точке/через киоск/в мобильном приложении без веб-Метрики.

Эти продажи **by design не попадут в сквозную** — у них нет веб-визита для привязки. Это не баг,
это граница инструмента. **Проговори с заказчиком** заранее: «сквозная покроет X% продаж (веб-заказы
с ClientID), остальные Y% (телефон/оффлайн) в ROI по кампаниям не войдут». Иначе позже будет
«почему сквозная показывает меньше продаж, чем в 1С» — а это ожидаемо.

## Проверка захвата (для /redanalyst-verify)

1. Открой сайт, создай тестовый заказ.
2. Проверь: hidden-поле заполнилось (DevTools) → значение в POST → значение в сохранённом заказе/CRM/1С.
3. Если на любом шаге пусто — там и рвётся; чини это звено, не гадай.
