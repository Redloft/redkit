# GA4 / GTM — вторичный стек (тонкий адаптер)

Основной стек скилла — Яндекс. Но концепции переносятся 1:1, поэтому для проектов на Google держим
тонкий адаптер за той же абстракцией «идентификатор + цель + загрузка», не дублируя логику.

## Маппинг Яндекс → Google

| Яндекс.Метрика | GA4 / GTM | Роль |
|---|---|---|
| ClientID | `client_id` (+ `session_id` для join в отчётах) | ключ склейки визит↔продажа |
| Offline conversions upload | **Measurement Protocol (GA4)** | серверная/офлайн доотправка событий |
| action-цель (reachGoal) | событие (`event`) в GA4 | целевое действие |
| ecommerce-цель | dataLayer `purchase` + `items[]` | выручка на сайте |
| Reporting API | **GA4 Data API** | чтение статистики |
| Directs расход/ROI | Google Ads link + import | нога «расход» |

## Ключевые эндпоинты (для справки, когда проект на Google)

- **Measurement Protocol (GA4)**: HTTP POST событий на `www.google-analytics.com/mp/collect` с
  `measurement_id` + `api_secret`; событие несёт `client_id` (обязателен для склейки с веб-сессией) и,
  рекомендуется, `session_id`. Это аналог офлайн-загрузки Метрики.
- **GTM dataLayer**: ecommerce-события (`purchase`, `add_to_cart`, `begin_checkout`) с массивом `items`.
- **GA4 Data API**: read-side отчётности (runReport).
- Google консолидирует приём данных в новый **Data Manager API** — держать в уме при долгих проектах.

## Отличия, о которых помнить

- В GA4 нет жёсткого «21-дневного окна офлайн-привязки» как в Метрике, но есть свои окна атрибуции и
  лаг обработки — так же verify по факту, не по HTTP 200.
- `client_id` в GA4 достаётся из cookie `_ga` (парсинг) или через `gtag('get', ...)` — асинхронно, те
  же грабли, что с `getClientID` (см. `clientid-capture.md`).
- 152-ФЗ так же применим (передача данных о покупке третьему лицу), плюс для Google — трансграничная
  передача, что усиливает комплаенс-требования. При RU-проекте на GA4 это отдельный правовой разговор.

Для большинства RU-проектов основной стек — Яндекс; GA4-адаптер держим тонким и разворачиваем только когда проект
реально на Google.
