# Methodology Proposals (MP) — {{PROJECT_NAME}}
<!-- EN: Self-improvement system. The methodology grows with the project via numbered proposals. (MP-029) -->

> Методология не статична — она растёт вместе с проектом. Заметил, что процесс можно улучшить,
> или правило мешает/устарело — оформи **MP** (Methodology Proposal). Так изменения процесса
> проходят через явное предложение, а не «по-тихому».
> _Process changes go through numbered proposals, not silent drift._

## Как это работает · How it works
1. Возникла идея по процессу (новое правило, изменение workflow, новая рутина) → создай
   `MP-NNN-краткое-название.md` по `MP-TEMPLATE.md` (следующий свободный номер — см. `INDEX.md`).
2. Статус MP: `proposed` → `accepted` → `applied` (или `rejected`).
3. При `applied` — внеси изменение в реальные доки (`HARD-RULES.md`, `START-HERE.md`, `tasks/PROTOCOL.md` и т.д.)
   и поставь `status: applied` + что именно поменялось.
4. Обнови `INDEX.md` (строка на каждый MP).

## Принцип · Principle
MP = «диф к процессу». Маленький, конкретный, с проблемой и предложением. Не манифест —
одна проблема = один MP. История MP = память, почему методология такая, какая есть.
