## Dispatcher — как распределяем работу · Routing
1. Новая работа? Определи направление (строка в реестре). Нет подходящего — заведи новое.
2. Каждое направление планируется в своём чате и кладёт задачи в общий `docs/tasks/` (по `PROTOCOL.md`).
3. Зависимость между направлениями → через `docs/chats/handoff-queue.md` (не через прямой пинг).
4. Общие правила и Hard Rules — едины для всех направлений (`docs/HARD-RULES.md`).

## Принцип · Principle
Одно направление = один контекст = один planning-чат. Не смешивай несвязанные направления в одном чате —
контекст раздувается, решения путаются.
