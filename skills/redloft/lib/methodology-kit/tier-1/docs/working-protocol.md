> Перед стартом классифицируй задачу: **Mode A** (стратегическая) или **Mode B** (быстрая).
> _Classify each task as Mode A (strategic) or Mode B (quick) before starting._

## Mode A — Strategic · Стратегическая
**Когда:** новая фича, рефактор нескольких модулей, изменение схемы БД, новая интеграция,
security-работа, что-то дольше пары часов.

**Поток:**
1. Research — прочитай код + `docs/tz.md`, собери контекст.
2. План — что меняем, какие файлы, риски. Запиши в задачу (`Что сделать` / `Готово когда`).
3. Self-review плана: понятен ли объём, есть ли критерии приёмки? (Если auth/схема/архитектура — `opus_review: true`, глубокий ревью.)
4. Реализация по плану.
5. Закрытие — `docs/prompts/iteration.md`.

## Mode B — Quick · Быстрая
**Когда:** мелкий фикс, текст, стиль, тривиальное изменение в одном файле.

**Поток:** делаешь → `/finalize` → коммит. Без отдельного планирования.

## Правило выбора · Rule
Сомневаешься A или B? → бери **A**. Недо-планирование дороже, чем лишние 5 минут плана.
