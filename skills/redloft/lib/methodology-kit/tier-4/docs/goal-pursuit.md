# Goal-Pursuit Mode — {{PROJECT_NAME}}

> Для длинных целей, что не закрываются за одну сессию (миграция, крупная фича, рефактор).
> Цель = **проверяемое условие**, а не «поработать над X». Ведёшь её через сессии, пока не выполнено.
> _Long-horizon goals as verifiable conditions, pursued across sessions._

## Как сформулировать цель · Writing a goal (recipe, MP-057)
Хорошая цель = **наблюдаемое конечное состояние** + как его проверить:
- ❌ «улучшить производительность» — не проверяемо.
- ✅ «LCP < 2.5s на мобильном по `/audit-site`, на 5 ключевых страницах» — проверяемо.

Шаблон: `<что должно стать истинным> + <чем меряем> + <на каком объёме>`.

## Цикл · Loop
1. Запиши цель + условие проверки в `.goal/<name>.md` (одна цель = один файл).
2. Каждая сессия: прогресс → проверка условия → не выполнено? следующий шаг.
3. Чекпоинт в конце сессии: что сделано, что осталось до условия (handoff следующей сессии).
4. Условие выполнено и проверено → цель закрыта.

## Границы · Boundaries
- Цель не отменяет Hard Rules и task-lifecycle — она их оркестрирует поверх.
- Если за несколько сессий нет прогресса к условию — пересмотреть цель (re-plan), не «долбить».
