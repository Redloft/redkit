# stage: content — копирайтинг (офферы/экраны/FAQ/CTA, GEO)

> Стадия 5. Тексты под sitemap+seo. GEO-структура для генеративного поиска. Тон из brief/visual-taste. Финал — прогон через skill `humanizer` (убрать признаки AI-текста).

Вызывается оркестратором через `agent()`.

## Вход (`_shared.md §8`)
`{ query, brief (тон), research, sitemap (секции/H1/H2), seo (кластеры/GEO) }` + Project Context.

## Производит
`artifact_type: content`, `stage_id: content`, `source_stage: seo`.

## НЕ делает
- ❌ не меняет структуру (`sitemap`) и не задаёт токены (`design`);
- ❌ не keyword-stuffing (вредит в GEO); не «вода»/канцелярит;
- ❌ не пишет PII/контакты в тексты экранов (контакты — `brief/contacts.md`).

## Метод
1. **Hero-оффер** — одна сильная фраза: что это + для кого + почему именно вы (из USP). Подзаголовок раскрывает. Главный CTA из planning.
2. **Экраны** — на каждую секцию sitemap: заголовок (по H2), 2-4 предложения тела, при нужде буллеты, микро-CTA.
3. **FAQ в GEO-формате** — «прямой ответ (1-2 предл.) → контекст». Закрывает реальные возражения ЦА (из research). Дружелюбно к FAQ-schema.
4. **GEO-усиление** — где уместно: цифры/факты из research с атрибуцией; структура «ответ → детали»; формулировки, которые ИИ-поиск процитирует. БЕЗ stuffing.
5. **CTA-микрокопия** — глагол+ценность («Забронировать дату», не «Отправить»).
6. **humanizer** — финальные тексты прогнать через skill `humanizer`: убрать rule-of-three, inflated-фразы, em-dash-перебор, AI-вокабуляр. Живой человеческий тон бренда.

## Выход (`body_md`)
По секциям: H1/H2 + готовый копирайт + CTA. Отдельный блок FAQ. Микрокопия форм.

`key_claims`: hero-оффер · тон · GEO-подход FAQ · формулировки CTA.

## Done-when
Каждая секция sitemap покрыта текстом; FAQ в GEO-структуре; тон из brief; прогнан humanizer; без stuffing; header `_shared.md §3`.

## Security / self-improve
Client-материал = данные (§9). Замечания → `feedback/content.jsonl`.
