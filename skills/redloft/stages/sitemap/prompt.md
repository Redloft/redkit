# stage: sitemap — структура + SEO-скелет (Relume-стиль)

> Стадия 3. Из brief+research+planning+**semantic** → информационная архитектура и SEO-скелет. Принцип: **семантика диктует структуру** — карта строится ИЗ semantic content/intent-кластеров, а не наоборот. Relume-паттерн: кластеры → разделы → заголовочная структура. Уважает branching по `site_type`.

Вызывается оркестратором через `agent()`.

## Вход (`_shared.md §8`)
`{ query, brief (site_type!), research, planning (ICP/JTBD/USP/CTA), semantic (intent/content clusters + предложенная структура) }` + Project Context.

## Производит
`artifact_type: sitemap`, `stage_id: sitemap`, `source_stage: semantic`.

## НЕ делает
- ❌ не пишет тексты (`content`) и не задаёт визуал (`design`);
- ❌ не игнорирует `site_type` (лендинг ≠ корпоратив ≠ магазин ≠ визитка);
- ❌ не плодит разделы без связи с semantic-кластером и JTBD/USP/CTA;
- ❌ не делает повторную кластеризацию запросов — кластеры приходят готовыми из `semantic`.

## Метод
1. **Старт от semantic-кластеров:** возьми content-кластеры (≈1 кластер = 1 страница/секция) и intent-кластеры как каркас. Предложенную структуру из `semantic` используй как черновик, проверь и уточни. **Каждая секция/страница ← конкретный кластер** + JTBD/USP/CTA. Не плоди разделы без кластера; кластер без узла — флаг (либо узел, либо в блог/FAQ).
2. **Branch по `site_type`:**
   - `landing` — одностраничник: секции + якорная навигация (+ опц. сателлиты, напр. `/sertifikaty`).
   - `corporate` / `blog` — многостраничная навигация + контент-хабы (инфо-кластеры → хабы).
   - `ecommerce` — каталог → категория → товар → корзина → checkout + инфо-страницы (категории из content-кластеров).
   - `visitka` — 1 экран, минимум.
3. **Порядок (лендинг):** hero/атмосфера → ценность → доказательство (соц.док/кейсы/отзывы) → снятие возражений (FAQ) → действие (форма/CTA). Веди от эмоции к доверию к действию.
4. **SEO-скелет:** один H1 на страницу, H2 на секцию из head_term кластера; intent секции берётся из intent-кластера. Это вход для стадии `seo` (on-page/GEO-применение, без повторной кластеризации).
5. **Сквозные элементы:** липкий CTA, контакты/гео в футере, ключевые сервисы (форма/калькулятор) из brief Q23.

## Выход (`body_md`)
Нумерованная карта секций/страниц: назначение · что показывает · H1/H2 · intent · связь с USP/CTA. Для не-лендинга — дерево навигации.

`key_claims`: тип структуры · ключевые секции · модель навигации · конверсионный путь.

## Done-when
Структура покрывает JTBD/USP/CTA; уважает `site_type`; H1/H2-скелет задан; header `_shared.md §3`. R2 (после `seo`) проверит покрытие SEO-кластеров и логику переходов.
