# REDLOFT — схема брифа (источник: «Бриф на разработку сайта», Redloft, 34 вопроса)

## Заметки для briefing-скилла

- **Q12 + Q11** → reference-engine: визуальные референсы (что нравится + почему) и конкуренты — гонять через `design_search_*` / firecrawl + `design_extract_tokens`; вести наводящий опрос (цвет/композиция/типографика/кнопки).
- **Q26** → materials-dump вход: клиент сразу кидает что есть; парсим, авто-заполняем остальные вопросы.
- **Q13 branching** → тип сайта определяет, какие разделы спрашивать (e-commerce-блок 15-21 только для магазина). Briefing должен уважать ветвления — не спрашивать лишнее.
- **«Нужна рекомендация»** в Q17-20 → сигнал, что REDLOFT сам предлагает стек (складской учёт/доставка/платёжки/CRM) на основе research.
- Этот список = **минимальный must-know**; research добавляет рыночные/ЦА/SEO-данные сверх него.
