# REDLOFT Website Builder System — SPEC (источник: бриф пользователя 2026-06-01)

> Статус: **фактуры и наброски**. Это исходное видение, зафиксированное дословно по смыслу. Архитектурный разбор — `ARCHITECTURE.md`. План MVP — `PLAN.md` (после research).

## Цель

Оркестратор, который проводит проект **от идеи до готового технического задания** на разработку сайта/лендинга. Использует существующие skills и запускает их последовательно, передавая результаты между этапами.

## Архитектура — 2 уровня

### Уровень 1 — Специализированные Skills (каждый только за свою область)

| Skill | Задачи | Результат |
|---|---|---|
| **Research** | бизнес, конкуренты, рынок, ЦА, лучшие практики, сбор материалов, база знаний проекта | Research Report |
| **Planning** | виртуальное агентство: CEO, Product Manager, UX, Marketing, SEO, Developer | Product Brief, ICP, JTBD, USP, структура проекта |
| **Sitemap** | структура сайта, логика разделов/переходов, SEO-структура | Sitemap |
| **SEO** | кластеризация запросов, структура SEO-страниц, рекомендации по контенту/мете | SEO Strategy |
| **Content** | контент, офферы, тексты экранов, FAQ, CTA, SEO-статьи | Content Pack |
| **Design** | визуальная концепция, moodboard, UI-рекомендации, дизайн-система, промты для Claude Code | Design Specification |
| **Performance** | скорость, техническая SEO-часть, Core Web Vitals | Technical Performance Report |
| **Reviewer** ⭐ | противоречия, логические ошибки, полнота, качество, критика предыдущих этапов | Review Report + список замечаний |

**Reviewer — самый важный skill.**

### Уровень 2 — Оркестраторы (не выполняют работу сами, управляют skills)

**Landing Builder** (MVP) — вход: описание бизнеса, URL, материалы клиента.
Пайплайн: Research → Planning → Sitemap → SEO → Content → Design → Reviewer.
Выход: готовое ТЗ, структура сайта, контент, дизайн-концепция, промт для реализации.

**Website Improver** — вход: существующий сайт.
Пайплайн: Research → SEO Audit → UX Audit → Content Audit → Reviewer.
Выход: список улучшений, приоритизация, roadmap.

**Product Builder** — вход: идея продукта.
Пайплайн: Research → Planning → Product → UX → Reviewer.
Выход: PRD, roadmap, MVP, архитектурная концепция.

## Принцип работы

Каждый следующий skill получает: (1) исходный запрос, (2) результаты предыдущих этапов, (3) замечания Reviewer. **Никакой этап не работает в изоляции.** Все артефакты — в **Project Context**.

## MVP = Landing Builder

Почему: проще тестировать; соответствует текущим задачам Redloft; быстрые реальные кейсы; фундамент для остальных оркестраторов.

## Критерий успеха

Пользователь пишет «Создай сайт для банного комплекса» → получает research + позиционирование + структуру + SEO-стратегию + тексты + дизайн-концепцию + ТЗ + промт для Claude Code, **без ручного запуска отдельных skills.**

## Дополнения пользователя (важно — учесть в плане)

1. **Memory Skill (будущее)** — не обычная память, а **память проекта**: после каждого запуска сохраняются исследования, решения, бренд-гайд, tone of voice, дизайн-система, SEO-кластеры. Второй запуск по тому же проекту **развивает накопленное**, не начинает с нуля. → «Именно этот слой отличает набор агентов от полноценной системы.»
2. **Документация** для разработчиков / клиентов / контент-менеджеров в красивом формате — референс https://dev.max.ru/help.
3. **Готовая база (reuse)** — уже есть работающий сайт на **Supabase + админка на Vercel/Beget**. Можно забрать принцип работы или поискать готовые сборки на гите.
4. **Уже есть skills** — `codegraph`, генерация контента (`content-gen`) — не забыть подключить.
5. **Performance → разделить** на **Performance + SEO + GEO** отдельно.
6. **Turnkey-передача** — продумать как передаём клиенту: перенос на его хостинг и БД.
7. Для проработки всего этого — **потестить redresearch** и составить план.
